## 1. Общая картина

### Задача одним предложением

Мы ищем **минимум функции** — точку, где функция принимает наименьшее значение.

### Что мы делаем в лабораторной

```
┌─────────────────────┐     ┌──────────────────┐     ┌────────────────────┐
│  Конструктивное      │     │  Чёрные ящики     │     │  Методы            │
│  число               │     │  (3 функции)      │     │  оптимизации       │
│                      │────▶│                    │────▶│                    │
│  Хранит число как    │     │  Принимают числа,  │     │  Ищут минимум      │
│  интервал [a, b]     │     │  возвращают        │     │  функции           │
│  с точностью ε=b-a   │     │  значение + градиент│    │                    │
└─────────────────────┘     └──────────────────┘     └────────────────────┘
```

**Зачем конструктивные числа?** В реальности мы не можем хранить числа с бесконечной точностью. Конструктивное число честно говорит: «я не знаю точное значение, но оно где-то в интервале [a, b]». Мы исследуем, как эта неточность (ε) растёт в процессе вычислений.

**Зачем три функции?** Чтобы сравнить, как методы ведут себя на «лёгкой» задаче, «сложной» задаче и «классически трудной» задаче.

**Зачем три метода?** Метод 0-го порядка не знает производных. Метод 1-го порядка знает первые производные. Метод 2-го порядка знает первые и вторые. Мы увидим, как дополнительная информация ускоряет поиск.

## 2. Конструктивное число

### Интуиция (ELI5)

Ты измеряешь длину стола рулеткой и получаешь «1.5 метра». Но рулетка не идеальна — реальная длина где-то между 1.49 и 1.51. Вместо того чтобы врать себе про «точно 1.5», мы честно храним **интервал [1.49, 1.51]**. Ширина интервала `ε = 1.51 - 1.49 = 0.02` — это наша **погрешность**.

### Формальное определение

**Конструктивное число** — пара `(a, b)`, где `a, b ∈ Q` (рациональные), `a ≤ b`.
Представляет действительное число, которое лежит где-то в интервале `[a, b]`.

- **Точность**: `ε = b - a`
- **Получение значения**: через параметр `α ∈ [0, 1]`: `x(α) = a·(1-α) + b·α`
  - `α = 0` → получаем `a` (левый край)
  - `α = 1` → получаем `b` (правый край)
  - `α = 0.5` → получаем середину `(a+b)/2`

### Два способа создания

```
Способ 1: Из пары (a, b)
  CN(1.0, 2.0) → интервал [1.0, 2.0], ε = 1.0

Способ 2: Из числа x и точности ε
  CN(5.0, epsilon=0.1) → интервал [4.95, 5.05], ε = 0.1
  (формула: a = x - ε/2, b = x + ε/2)
```

### Интервальная арифметика — правила

Пусть `x = [a₁, b₁]` и `y = [a₂, b₂]`.

#### Сложение
```
[a₁, b₁] + [a₂, b₂] = [a₁ + a₂, b₁ + b₂]
```
**Ширина результата** = ε₁ + ε₂ (точности складываются).

*Пример*: `[1, 2] + [3, 4] = [4, 6]`, ширина: 1 + 1 = 2.

#### Вычитание
```
[a₁, b₁] - [a₂, b₂] = [a₁ - b₂, b₁ - a₂]
```
**Ширина результата** = ε₁ + ε₂ (тоже складываются!).

*Пример*: `[3, 5] - [1, 2] = [3-2, 5-1] = [1, 4]`, ширина: 2 + 1 = 3.

**Почему a₁ - b₂?** Чтобы получить наименьший возможный результат, берём наименьшее из первого и вычитаем наибольшее из второго.

#### Умножение
```
[a₁, b₁] × [a₂, b₂] = [min(все произведения), max(все произведения)]
```
Все произведения = `{a₁·a₂, a₁·b₂, b₁·a₂, b₁·b₂}`.

*Пример*: `[2, 3] × [4, 5]`: произведения = {8, 10, 12, 15} → `[8, 15]`.

*С отрицательными*: `[-1, 2] × [3, 4]`: произведения = {-3, -4, 6, 8} → `[-4, 8]`.

**Ширина растёт** как `ε₁·|y| + ε₂·|x| + ε₁·ε₂` — то есть пропорционально величинам чисел.

#### Деление
```
[a₁, b₁] / [a₂, b₂] = [a₁, b₁] × [1/b₂, 1/a₂]
```
**Важно**: если `0 ∈ [a₂, b₂]`, деление невозможно (деление на ноль).

*Пример*: `[4, 6] / [2, 3] = [4, 6] × [1/3, 1/2]`. Произведения: {4/3, 2, 2, 3} → `[4/3, 3]`.

#### Возведение в квадрат (оптимизация!)
Для `x² = [a, b]²`:
- Если `a ≥ 0`: `[a², b²]` (оба положительны — квадрат сохраняет порядок)
- Если `b ≤ 0`: `[b², a²]` (оба отрицательных — квадрат переворачивает порядок)
- Если `a < 0 < b`: `[0, max(a², b²)]` (ноль входит, минимум квадрата = 0)

### Ключевой факт: ε всегда РАСТЁТ

```
Операция       │ Как растёт ε
───────────────┼──────────────────────────────
Сложение       │ ε₁ + ε₂ (линейно)
Вычитание      │ ε₁ + ε₂ (линейно)
Умножение      │ ~ ε₁·|y| + ε₂·|x| (быстро)
Деление        │ ещё быстрее
Цепочка операций│ ЭКСПОНЕНЦИАЛЬНО!
```

**За 100 итераций** градиентного спуска ε может вырасти на 10-20 порядков. Это **не баг, а фича** — мы это и исследуем в лабораторной.

### Проблема зависимости (dependency problem)

Классическая ловушка интервальной арифметики:
```
x = [1, 2]
x - x = ?
```
- **Правильный ответ**: 0 (мы же вычитаем число из самого себя)
- **Ответ интервальной арифметики**: `[1-2, 2-1] = [-1, 1]`

Почему? Интервальная арифметика **не знает**, что оба операнда — одно и то же число. Она трактует каждое вхождение переменной как **независимое**. Это главный источник завышения ε.

## 3. Python: перегрузка операторов

### Зачем это нужно

Чтобы вместо `c1.add(c2)` писать `c1 + c2`. Python позволяет определить, что означают `+`, `-`, `*`, `/`, `<`, `>` и т.д. для твоего класса.

### Как это работает

Когда Python видит `a + b`, он делает:
1. Пробует вызвать `a.__add__(b)`
2. Если тот вернул `NotImplemented`, пробует `b.__radd__(a)`

### Шпаргалка: все нужные методы

```python
class MyNumber:
    def __init__(self, value):
        self.value = value

    # === Арифметика ===

    def __add__(self, other):       # self + other
        ...
    def __radd__(self, other):      # other + self (когда other не знает про нас)
        ...
    def __sub__(self, other):       # self - other
        ...
    def __rsub__(self, other):      # other - self  ⚠️ НЕ коммутативно!
        ...
    def __mul__(self, other):       # self * other
        ...
    def __rmul__(self, other):      # other * self
        ...
    def __truediv__(self, other):   # self / other
        ...
    def __rtruediv__(self, other):  # other / self
        ...
    def __pow__(self, n):           # self ** n
        ...
    def __neg__(self):              # -self (унарный минус)
        ...

    # === Сравнение ===

    def __eq__(self, other):        # self == other
        ...
    def __lt__(self, other):        # self < other
        ...
    def __le__(self, other):        # self <= other
        ...
    def __gt__(self, other):        # self > other
        ...
    def __ge__(self, other):        # self >= other
        ...

    # === Представление ===

    def __str__(self):              # print(self), str(self)
        ...
    def __repr__(self):             # repr(self), отображение в консоли
        ...
    def __float__(self):            # float(self) — приведение к float
        ...
```

### Пример: почему нужен `__radd__`

```python
a = CN(1.0, 2.0)  # наше конструктивное число

# Это работает:
result = a + 5.0   # вызывает a.__add__(5.0) ✓

# А это без __radd__ сломается:
result = 5.0 + a   # Python пробует (5.0).__add__(a) → int не знает про CN!
                    # Потом пробует a.__radd__(5.0) → ✓ если определён
```

### Важно: `__rsub__` НЕ коммутативен!

```python
def __rsub__(self, other):
    # Вызывается при: other - self  (не self - other!)
    # Нельзя просто вернуть self.__sub__(other)
    o = self._to_cn(other)
    return CN(o.a - self.b, o.b - self.a)
```

### isinstance — проверка типа

```python
def __add__(self, other):
    if isinstance(other, ConstructiveNumber):
        return CN(self.a + other.a, self.b + other.b)
    if isinstance(other, (int, float)):
        return CN(self.a + other, self.b + other)
    raise TypeError(f"Нельзя сложить CN с {type(other)}")
```

## 4. Число обусловленности

### Интуиция (ELI5)

Представь резиновый лист с нарисованной сеткой. Матрица — это преобразование, которое **растягивает и поворачивает** этот лист. Число обусловленности `κ` (каппа) — это **отношение максимального растяжения к минимальному**.

```
κ = 1:    квадраты → квадраты (равномерно)     ■ → ■
κ = 100:  квадраты → тонкие полоски             ■ → ═══
```

### Формула

Для симметричной положительно определённой матрицы A:

```
κ(A) = λ_max / λ_min
```

где `λ_max` и `λ_min` — наибольшее и наименьшее **собственные значения** матрицы.

### Как влияет на оптимизацию

Число обусловленности определяет **форму линий уровня** функции:

```
κ ≈ 1:                κ = 100:
    ●                      ╔══════════╗
   ╱ ╲                     ║          ║
  ╱   ╲                    ║    ●     ║
  ╲   ╱                    ║          ║
   ╲ ╱                     ╚══════════╝

Круглые линии уровня    Вытянутые эллипсы
Градиент → прямо к      Градиент → "зигзаги"
минимуму                 вдоль стенок долины
Быстрая сходимость       Медленная сходимость
```

### Влияние на градиентный спуск

Ошибка за одну итерацию уменьшается в `(κ-1)/(κ+1)` раз:

| κ | Множитель за шаг | Итераций для ε=10⁻⁶ |
|---|---|---|
| 1.16 | 0.07 | ~5 |
| 10 | 0.82 | ~70 |
| 100 | 0.98 | ~700 |
| 1000 | 0.998 | ~7000 |

**Метод Ньютона не зависит от κ** — это его главное преимущество.

## 5. Целевые функции

### Функция 1: Квадратичная с хорошим κ (6 переменных)

```
f(x) = ½ xᵀAx + bᵀx + c
```

где `A` — матрица 6×6 с собственными значениями `{0.95, 0.98, 1.0, 1.02, 1.05, 1.1}`.

- **Число обусловленности**: κ ≈ 1.16 (почти идеальная задача)
- **Минимум**: x* = -A⁻¹b (формула из линейной алгебры)
- **Градиент**: ∇f(x) = Ax + b
- **Гессиан**: H = A (постоянный!)

*Почему лёгкая?* Линии уровня почти круглые → градиент указывает почти прямо на минимум.

### Функция 2: Квадратичная с плохим κ (4 переменных)

Та же формула, но `A` — матрица 4×4 с собственными значениями `{1, 10, 50, 100}`.

- **Число обусловленности**: κ = 100
- **Минимум**: x* = -A⁻¹b
- **Градиент и гессиан**: те же формулы

*Почему сложная?* Линии уровня — сильно вытянутые эллипсы → градиентный спуск «зигзагит».

### Функция 3: Розенброка (3 переменных)

```
f(x) = Σᵢ [100·(xᵢ₊₁ - xᵢ²)² + (1 - xᵢ)²]
```

- **Минимум**: x* = (1, 1, 1), f(x*) = 0
- **Градиент**:
  ```
  ∂f/∂xᵢ = -400·xᵢ·(xᵢ₊₁ - xᵢ²) + 2·(xᵢ - 1)    (для i < n)
  ∂f/∂xₙ = 200·(xₙ - xₙ₋₁²)                        (для последнего)
  ```
- **Гессиан**: трёхдиагональная матрица
  ```
  Hᵢᵢ = -400·(xᵢ₊₁ - 3xᵢ²) + 2
  Hᵢ,ᵢ₊₁ = Hᵢ₊₁,ᵢ = -400·xᵢ
  Hₙₙ = 200
  ```

### Почему Розенброк — классический тест

```
Вид сверху на линии уровня (2D):

  y ↑
    │    ╭─────────╮
    │   ╱           ╲      ← узкая изогнутая "банановая" долина
    │  ╱  ○ минимум  │
    │  ╲    (1,1)   ╱
    │   ╲           ╱
    │    ╰─────────╯
    └──────────────────→ x
```

**Найти долину легко** — стенки крутые, функция быстро падает.
**Дойти по долине до минимума — очень сложно** — дно почти плоское и изогнутое.

Градиент на дне долины **почти перпендикулярен** направлению к минимуму → градиентный спуск ползёт крайне медленно.

## 6. Нелдер–Мид (метод 0-го порядка)

### Интуиция (ELI5)

Ты и двое друзей потерялись в горах в тумане. Компаса нет (нет градиента). Стратегия: встаньте треугольником. Кто стоит выше всех — **перепрыгивает через остальных** на противоположную сторону, надеясь оказаться ниже. Треугольник «ползёт» вниз, деформируясь по дороге.

### Алгоритм по шагам

Для функции `n` переменных строится **симплекс** из `n+1` вершин.

```
                    Шаг 1: СОРТИРОВКА
     Упорядочить вершины по f:
     f(x₁) ≤ f(x₂) ≤ ... ≤ f(xₙ₊₁)
     x₁ = лучшая, xₙ₊₁ = худшая

                    Шаг 2: ЦЕНТРОИД
     x_c = среднее всех точек КРОМЕ худшей

                    Шаг 3: ОТРАЖЕНИЕ
     xᵣ = x_c + α·(x_c - xₙ₊₁)     α = 1
     (отражаем худшую точку через центроид)

        ┌──────────────────────────────────────┐
        │  Если f(x₁) ≤ f(xᵣ) < f(xₙ):       │
        │    → ПРИНЯТЬ отражение               │
        │                                      │
        │  Если f(xᵣ) < f(x₁):                │
        │    → Попробовать РАСТЯЖЕНИЕ          │
        │    xₑ = x_c + γ·(xᵣ - x_c)  γ = 2  │
        │    Взять лучшее из xᵣ и xₑ          │
        │                                      │
        │  Если f(xᵣ) ≥ f(xₙ):                │
        │    → Попробовать СЖАТИЕ              │
        │    x_s = x_c + ρ·(xₙ₊₁ - x_c) ρ=0.5│
        │    Если помогло → принять            │
        │    Если нет → ГЛОБАЛЬНОЕ СЖАТИЕ      │
        │    Все xᵢ → x₁ + σ·(xᵢ - x₁) σ=0.5 │
        └──────────────────────────────────────┘
```

### Визуализация операций (2D)

```
  Исходный                 Отражение            Растяжение
  симплекс

      ×₃(худшая)                                       ×ₑ
       ╱╲                      ╱╲                      ╱╲
      ╱  ╲                    ╱  ╲                    ╱  ╲
    ×₁────×₂               ×₁────×₂               ×₁────×₂
                              ╲  ╱
                               ╲╱
                                ×ᵣ

  Сжатие                   Глобальное сжатие

       ×_s
       ╱╲                      ×₁
      ╱  ╲                    ╱╲
    ×₁────×₂                ×₃'──×₂'
                            (все сжались к лучшей)
```

### Плюсы и минусы

| ✓ Плюсы | ✗ Минусы |
|---|---|
| Не нужны производные | Медленная сходимость |
| Прост в реализации | Плохо в высоких размерностях (>20) |
| Работает с шумными функциями | Нет гарантий сходимости |
| Устойчив к негладкости | Может «застрять» |

## 7. Градиентный спуск (метод 1-го порядка)

### Интуиция (ELI5)

Ты на горе в тумане, но **чувствуешь наклон** под ногами. Стратегия: шагай туда, куда наклон самый крутой вниз. Размер шага — **learning rate** (η).

### Формула

```
x_{k+1} = x_k - η · ∇f(x_k)
```

- `∇f(x_k)` — градиент (вектор частных производных) в текущей точке
- `η` — learning rate (скорость обучения / длина шага)
- Знак «минус» — идём **против** градиента (вниз, а не вверх)

### Learning rate: как подобрать

```
η слишком большой:          η правильный:          η слишком маленький:

    ╱╲    ╱╲                     ╲                        ╲
   ╱  ╲  ╱  ╲  → расходится       ╲                       ╲
  ╱    ╲╱    ╲                      ╲  → сходится быстро    ╲. . . . → ползёт
 ╱              ╲                    ╲╱                       ╲___
                                     ● минимум                     __●
```

**Теоретический оптимум**: `η = 2 / (λ_max + λ_min)`, где λ — собственные значения гессиана.

**Практическое правило**:
- `η < 2/λ_max` — иначе расходится
- Для квадратичной функции с κ ≈ 1: η можно взять большим (~1)
- Для κ = 100: η ≈ 0.01–0.02
- Для Розенброка: η ≈ 0.001–0.003

**В лабораторной** нужно **подобрать** оптимальный η: попробовать несколько значений и выбрать лучший.

### Скорость сходимости

Для сильно выпуклой функции:

```
||x_k - x*|| ≤ ((κ-1)/(κ+1))^k · ||x₀ - x*||
```

Это **линейная сходимость** — каждая итерация уменьшает ошибку в фиксированное число раз.

| κ | Множитель | Характер |
|---|---|---|
| 1.16 | 0.07 | Очень быстро |
| 100 | 0.98 | Очень медленно |

### Как работает с конструктивными числами

```python
# Начало: x = [CN(0, epsilon=1e-5), CN(0, epsilon=1e-5), ...]
# Итерация 1: g = grad(x)  — уже интервалы, ε вырос
#             x = x - η·g  — ε вырос ещё
# Итерация 2: g = grad(x)  — ε растёт экспоненциально
# ...
# Итерация 100: ε может быть 10^10 и больше!
```

ε растёт потому, что **каждая итерация** — это цепочка операций, каждая из которых расширяет интервал. И следующая итерация берёт на вход уже расширенный интервал.

## 8. Метод Ньютона (метод 2-го порядка)

### Интуиция (ELI5)

Градиентный спуск чувствует только **наклон** (1-я производная). Метод Ньютона чувствует ещё и **кривизну** (2-я производная). Он строит в текущей точке **параболу** (квадратичное приближение функции) и прыгает **сразу в минимум этой параболы**.

```
    f(x)
    │
    │  ○ текущая точка
    │ ╱╲
    │╱  ╲  ← реальная функция
    │    ╲
    │  ╱  ╲╱  ← параболическое приближение
    │ ╱    ●  ← минимум параболы = следующая точка
    │╱
    └──────────── x
```

Если функция **действительно квадратичная**, парабола совпадает с функцией → минимум находится за **один шаг**!

### Формула

```
x_{k+1} = x_k - H⁻¹(x_k) · ∇f(x_k)
```

- `H(x_k)` — **гессиан** (матрица вторых производных)
- `H⁻¹` — обратная матрица (на практике решаем СЛАУ `H·Δx = -∇f`)

### Гессиан — что это

Матрица всех вторых частных производных:

```
        ┌ ∂²f/∂x₁²    ∂²f/∂x₁∂x₂  ...  ∂²f/∂x₁∂xₙ ┐
H(x) =  │ ∂²f/∂x₂∂x₁  ∂²f/∂x₂²    ...  ∂²f/∂x₂∂xₙ │
        │ ...           ...          ...  ...          │
        └ ∂²f/∂xₙ∂x₁  ∂²f/∂xₙ∂x₂  ...  ∂²f/∂xₙ²   ┘
```

Для **квадратичной функции** `f = ½xᵀAx + bᵀx + c`: гессиан `H = A` (постоянный!).

### Квадратичная сходимость

Вблизи минимума:
```
||x_{k+1} - x*|| ≤ C · ||x_k - x*||²
```

Это значит: если ошибка = 10⁻³, следующая будет ~10⁻⁶, потом ~10⁻¹², потом ~10⁻²⁴. Невероятно быстро.

### Сравнение с градиентным спуском

| | Градиентный спуск | Ньютон |
|---|---|---|
| **Использует** | Градиент | Градиент + гессиан |
| **Сходимость** | Линейная: `ошибка × 0.98` | Квадратичная: `ошибка²` |
| **Стоимость шага** | O(n) | O(n³) — решение СЛАУ |
| **Зависит от κ?** | Да, сильно | Нет! |
| **На квадратичной** | ~30 итераций (κ≈1) | **1 итерация** |
| **Learning rate** | Нужно подбирать | Не нужен (шаг = 1) |
| **Может расходиться?** | При большом η | Далеко от минимума |

## 10. Что исследовать и какие графики строить

### Обязательные графики

1. **Подбор learning rate**: графики `|f(x_k) - f(x*)| vs итерация` для разных η → выбрать лучший

2. **Сходимость по итерациям**: для каждой функции — 3 кривые (NM, GD, Newton) на одном графике. Ось Y — логарифмическая.

3. **Траектории на линиях уровня**: 2D-сечение (первые две координаты). Контурный график функции + пути всех трёх методов.

4. **Динамика ε**: как средний ε растёт по итерациям для GD и Newton при трёх начальных ε₀ = {10⁻², 10⁻⁵, 10⁻⁸}.

5. **Покомпонентная динамика ε**: как ε каждой координаты растёт отдельно.

### Какие числа записать

| Что | Зачем |
|---|---|
| Число итераций | Скорость сходимости |
| Число вызовов f | Реальная стоимость |
| Число вызовов ∇f | Стоимость по производным |
| `‖x - x*‖` | Качество результата |
| Время (секунды) | Практическая скорость |
| Финальный ε / начальный ε | Рост погрешности |

## 11. Подводные камни

### При реализации CN
- `__rsub__` не коммутативен — `a - b ≠ b - a`
- Не забыть `__radd__`, `__rmul__` — иначе `5.0 + cn` сломается
- Деление на интервал с нулём → исключение
- При `x²` оптимизировать отдельно (не через `x * x`) — иначе ε завышается из-за проблемы зависимости

### При реализации чёрных ящиков
- Проверять `isinstance(x[0], ConstructiveNumber)` для выбора ветки вычислений
- Не забыть считать вызовы (`func_calls`, `grad_calls`)
- Матрицу A создавать через `Q·Λ·Qᵀ` с заданными собственными значениями

### При реализации оптимизаторов
- Градиентный спуск: при слишком большом η получите `nan` или `inf`
- Ньютон: `np.linalg.solve(H, g)` лучше, чем `np.linalg.inv(H) @ g`
- Нелдер–Мид: не забыть глобальное сжатие (shrink) — без него может зацикливаться
- Критерий остановки: `‖∇f‖ < tol` для GD/Newton, `std(f_values) < tol` для NM

### При исследовании
- Для Розенброка начальная точка **не ноль** — лучше `(-1, -1, -1)`, чтобы было интереснее
- ε растёт экспоненциально — **это нормально**, это и нужно показать
- Не забыть `reset_counters()` перед каждым запуском метода
