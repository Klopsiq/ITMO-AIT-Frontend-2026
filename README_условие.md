# Лабораторная работа #1

Глобальная задача реализовать тип данных "конструктивное число" и несколько методов оптимизации целевой вычислимой функции и всё это потестировать.

## Блок 1 (Конструктивное число)

Реализовать тип данных "конструктивное число".

Данный тип данных состоит из двух чисел $a,b\in\mathbb{Q}$.

Данный тип должно быть возможно создать, имея:

1) Число $x\in\mathbb{R}$ и $\varepsilon\in\mathbb{R}$
2) Пара чисел $a,b\in\mathbb{Q}$

Данный тип данных должно быть возможно:

1) Складывать с себе подобным и действительным числом
2) Вычитать из себе подобного и действительного числа
3) Умножать на себя подобное и действительное число
4) Делить на себе подобное и действительное число
5) Сравнивать к себе подобным
6) Получать обратно желаемое действительное число

> Необходимо, чтобы вызовы сложения/вычитания/т.д. были нативными (естественными, пример: $a + b$).
>
> Желаемое действительное число предполагается получать через параметр $\alpha\in[0,1]$, где при $\alpha=0$ получаем $a$, а при $\alpha=1$ получаем $b$.

## Блок 2 (Черный ящик)

Реализовать ящики функций, таким образом чтобы у них был унифицированный интерфейс вызова.

В качестве ящиков будем использовать три функции:

1) Квадратичная функция 6-арная с хорошим числом обусловленности (около 1)

2) Квадратичная функция 4-арная с плохим числом обусловленности (около 100)

3) Функция Розенброка 3-арная.

> Стоит учитывать, что вероятно потребуется хранить информацию о производных
>
> Держим в голове, что черные ящики должны быть совместимы с "Конструктивным числом"

## Блок 3 (Методы оптимизации)

Реализовать методы оптимизации 0-го, 1-го порядка.

Метод 0-го порядка, как пример метод деформируемого многогранника, но можно и свой.

Метод 1-го порядка градиентный спуск.

## Блок 4 (Исследование)

Необходимо исследовать скорость и качество сходимости методов поиска минимума. Определить насколько близко методы подходят к минимуму; сколько необходимо итераций/времени; какое количество было вызовов функци (или производной). Построить траекторию движения спуска по линиям уровня (взять сечение или две оси).

> Стоит учитывать, что для градиентного спуска необходимо **подобрать** оптимальный `learning rate`.

Также, так как мы работае с "конструктивным числом" необходимо провести исследования по тому как увеличивается $\varepsilon$ при поиске минимума (построить графики отражающие динамику $\varepsilon$). Сравнить на разных функция и методах полученные $\varepsilon$. И в целом провести исследования на разных $\varepsilon$ (взять три разных порядка $\varepsilon$).
